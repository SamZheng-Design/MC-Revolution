### 关键设计要点
1. **身份通是统一入口**：所有用户通过身份通认证后，按角色分流
2. **融资者只需上传数据**：通过申请通整理/上传信息，数据进入投资者筛选池
3. **评估通+风控通是投资者的AI筛子**：每个投资者可自定义投资标准和核验方式
4. **机会通是投资者统一看板**：展示筛后项目，不设筛子=看到全部项目
5. **条款通是Y型汇合点**：投融资双方开始协同

## 页面路由

### 主页面
| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | 重定向 | 自动跳转到 `/design` |
| `/design` | 产品设计思路 | Y型业务流程图、架构总览（5列分组）、设计思路手风琴（5个模块含AI筛子详解） |
| `/portal` | 产品统一入口 | Y型分流布局、双角色路径展示、阶段化产品卡片 |

### 产品占位页面（9个"通"）
| 路径 | 产品名 | 角色归属 | 阶段 |
|------|--------|---------|------|
| `/identity` | 身份通 | 统一入口 | 入口 |
| `/application` | 申请通 | 融资者 | 数据上传 |
| `/assess` | 评估通 | 投资者 | AI筛子 |
| `/risk` | 风控通 | 投资者 | AI筛子 |
| `/opportunity` | 机会通 | 投资者 | 看板展示 |
| `/terms` | 条款通 | 双方协同 | 交易达成 |
| `/contract` | 合约通 | 双方协同 | 交易达成 |
| `/settlement` | 结算通 | 双方协同 | 投后管理 |
| `/performance` | 履约通 | 双方协同 | 投后管理 |

## 数据架构
- **数据源**: 静态TypeScript数据层 (`src/data.ts`)
- **产品角色分类**: shared / borrower / investor / collaborative
- **产品阶段分类**: entry / borrower-upload / investor-filter / investor-view / deal / post-investment
- **底座数据**: Account（含角色分流）、Data（含标准化）、AI（含筛子编排）
